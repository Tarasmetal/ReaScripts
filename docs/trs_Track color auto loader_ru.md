## **📖 Описание**
**Track Color Auto Loader & Gradients** — это мощный интеллектуальный инструмент для автоматизации визуальной организации вашего проекта в REAPER. Скрипт позволяет мгновенно окрашивать треки на основе их имен, используя гибкую систему правил, а также создавать профессиональные градиенты для содержимого папок.

Это избавляет от рутинной ручной раскраски, делая навигацию в сложных проектах быстрой и интуитивно понятной.

---

## **✨ Ключевые возможности**
- **Автоматизация по именам:** Раскраска треков по префиксам, суффиксов или вхождению текста в имя.
- **Продвинутые градиенты:** Создание плавных цветовых переходов для треков внутри папок (поддержка 2-х и 3-х точечных градиентов).
- **Тип правила "Receive":** Уникальная возможность окрашивать трек в зависимости от того, с какого трека на него приходит сигнал (идеально для шин обработки).
- **Система пресетов:** Сохранение различных цветовых схем для разных задач (запись, сведение, саунд-дизайн) и быстрое переключение между ними.
- **Интеграция с контекстом:** Копирование цвета выбранного трека прямо в палитру правила через правый клик.
- **Headless режим:** Возможность запуска скрипта без открытия окна (применение цветов и мгновенный выход).
- **Undo-безопасность:** Полная поддержка истории отмены REAPER.

---

## **🛠️ Как это работает**
1. **Анализ имен:** Скрипт сканирует все треки в проекте и ищет совпадения в именах согласно вашему списку правил.
2. **Приоритеты:** Правила префиксов и суффиксов имеют приоритет над общими правилами папок, что позволяет гибко настраивать исключения внутри градиентов.
3. **Расчет градиента:** Если трек является папкой и для него задано правило `Folder`, скрипт автоматически распределяет цвета между всеми дочерними треками, создавая равномерный переход от начального к конечному (и опционально среднему) цвету.
4. **Связи (Receives):** При использовании типа `Receive`, скрипт проверяет источники посылов на данный трек и ищет совпадения имен среди них.

## **🚀 Использование**

### **Шаг 1: Создание правил**
1. Нажмите **"Add Rule"** для создания нового правила.
2. Выберите тип:
    - `Prefix`: поиск совпадения в начале имени.
    - `Suffix`: поиск совпадения в конце имени.
    - `Folder`: создание градиента для содержимого папки.
    - `Receive`: окраска по имени источника посыла.
3. Введите текст (Pattern), который скрипт должен искать в имени трека.

### **Шаг 2: Выбор цветов**
- Нажмите на квадрат цвета, чтобы выбрать его вручную.
- **Лайфхак:** Нажмите правой кнопкой мыши по квадрату цвета, чтобы мгновенно скопировать цвет активного (выбранного) трека из вашего проекта.

### **Шаг 3: Применение**
Нажмите **"Run / Apply Colors"**. Скрипт мгновенно окрасит все подходящие треки в проекте. Если включена модификация, правила автоматически сохранятся в текущий пресет.

### **Шаг 4: Работа с пресетами**
Используйте меню **"File"** для сохранения (`Save`), создания копий (`Save As`) или загрузки существующих наборов правил.
